## 2. 身份认证设计

### 2.1 用户认证

#### 2.1.1 认证方式

| 认证方式 | 安全等级 | 适用场景 | 说明 |
|---------|---------|---------|------|
| 用户名+密码 | 中 | 日常登录 | 基础认证 |
| 短信验证码 | 中高 | 敏感操作 | 二次验证 |
| LDAP/AD集成 | 高 | 企业内部员工 | 单点登录 |
| 双因素认证(2FA) | 极高 | 管理员/敏感操作 | 推荐启用 |

#### 2.1.2 密码策略

| 规则 | 要求 |
|-----|------|
| 密码长度 | 最少8位 |
| 字符要求 | 大小写字母 + 数字 + 特殊字符 |
| 复杂度 | 不能包含用户名 |
| 定期更换 | 90天强制更换 |
| 历史记录 | 不能重复使用最近5个密码 |
| 错误锁定 | 连续5次错误锁定30分钟 |

#### 2.1.3 JWT Token设计

```json
{
  "header": {
    "alg": "RS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "user_id",
    "username": "admin",
    "roles": ["admin", "product_manager"],
    "permissions": ["product:read", "product:write"],
    "iat": 1705312200,
    "exp": 1705315800,
    "iss": "derucci-iot-platform"
  },
  "signature": "..."
}
```

**Token配置**：
| 参数 | 值 | 说明 |
|-----|------|------|
| 算法 | RS256 | 非对称加密，更安全 |
| 有效期 | 1小时 | 访问令牌 |
| 刷新令牌 | 7天 | 长期令牌 |
| 签发者 | derucci-iot-platform | 平台标识 |

### 2.2 设备认证

#### 2.2.1 认证方式对比

| 认证方式 | 安全等级 | 适用场景 | 密钥长度 |

#### 2.2.3 密钥生成规则

| 密钥 | 格式 | 长度 | 示例 |
|-----|------|------|------|
| ProductKey | PK + 随机字符 | 20 | PK7k2d9s8a1b5c3e6f |
| DeviceKey | DK + 随机字符 | 20 | DK3m7n1p4q8r2t5u9v |
| DeviceSecret | 随机字符串 | 32 | aB3cD5eF7gH9iJ1kL3m |

### 2.3 API认证

#### 2.3.1 认证流程

```
1. 用户登录 → 获取 Access Token
2. 携带 Token 请求 API: Authorization: Bearer <token>
3. Gateway 验证 Token 有效性
4. 验证通过 → 转发到后端服务
5. 验证失败 → 返回 401 Unauthorized
```

#### 2.3.2 认证失败处理

| 错误码 | 错误信息 | 处理方式 |
|-------|---------|---------|
| 40101 | Token无效 | 提示重新登录 |
| 40102 | Token过期 | 提示重新登录 |
| 40103 | 无权限访问 | 返回403 |
| 40104 | 账号已被禁用 | 提示联系管理员 |

## 4. 数据安全设计

### 4.1 数据分类分级

#### 4.1.1 数据分级

| 级别 | 定义 | 示例 | 保护要求 |
|-----|------|------|---------|
| L4-极敏感 | 核心资产 | 用户密码, 设备密钥 | 强加密+严格访问 |
| L3-敏感 | 重要数据 | 个人隐私, 业务数据 | 加密+审计 |
| L2-内部 | 内部数据 | 产品信息, 日志 | 访问控制 |
| L1-公开 | 可公开数据 | 设备型号, 固件信息 | 基础保护 |

#### 4.1.2 敏感数据清单

| 数据类别 | 字段示例 | 级别 | 加密方式 |
|---------|---------|-----|---------|
| 用户凭证 | password | L4 | BCrypt加密 |
| 设备密钥 | device_secret | L4 | AES-256 |
| 个人信息 | phone, email | L3 | AES-256 |
| 业务数据 | product, device | L2 | 访问控制 |
| 日志数据 | operate_log | L2 | 脱敏处理 |

### 4.2 存储加密

#### 4.2.1 数据库加密

**方案一：透明数据加密(TDE)**
```sql
-- MySQL TDE配置
ALTER TABLE sensitive_data ENCRYPTED=YES;
```

**方案二：应用层加密**
```java
// 使用AES-256加密敏感字段
public String encrypt(String plainText) {
    Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
    SecretKeySpec keySpec = new SecretKeySpec(getSecretKey(), "AES");
    cipher.init(Cipher.ENCRYPT_MODE, keySpec);
    return Base64.encode(cipher.doFinal(plainText.getBytes()));
}
```

#### 4.2.3 文件加密

```java
// 固件文件签名
public String signFirmware(byte[] content) {
    Signature signature = Signature.getInstance("SHA256withRSA");

}
```

## 7. 应急响应设计

### 7.1 安全事件分级

| 级别 | 定义 | 示例 | 响应时间 |
|-----|------|------|---------|
| P0-紧急 | 数据泄露/服务瘫痪 | 数据库被拖库 | 15分钟 |
| P1-严重 | 核心功能受损 | 被植入后门 | 1小时 |
| P2-重要 | 非核心功能受损 | 账号被盗 | 4小时 |
| P3-一般 | 轻微安全事件 | 异常登录 | 24小时 |

## 10. 总结

### 10.1 安全建设原则

1. **纵深防御**：多层防护，无单点安全
2. **最小权限**：按需授权，权限最小化
3. **零信任**：不信任，持续验证
4. **持续监控**：实时监控，快速响应
