## 2. 核心表结构

### 2.1 产品表 (product)

```sql
CREATE TABLE `product` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `name` VARCHAR(100) NOT NULL COMMENT '产品名称',
  `category` VARCHAR(50) NOT NULL COMMENT '产品分类: bed/pillow/monitor/other',
  `description` TEXT COMMENT '产品描述',
  `images` JSON COMMENT '产品图片URL列表',
  `thing_model_id` BIGINT COMMENT '关联物模型ID',
  `firmware_id` BIGINT COMMENT '默认固件ID',
  `device_count` INT DEFAULT 0 COMMENT '设备数量',
  `status` VARCHAR(20) NOT NULL DEFAULT 'draft' COMMENT '状态: draft/published/unpublished',
  `remark` VARCHAR(500) COMMENT '备注',
  `created_by` BIGINT NOT NULL COMMENT '创建人ID',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
  PRIMARY KEY (`id`),
  INDEX `idx_category` (`category`),
  INDEX `idx_status` (`status`),
  INDEX `idx_thing_model_id` (`thing_model_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='产品表';
```

### 2.2 物模型表 (thing_model)

```sql
CREATE TABLE `thing_model` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `name` VARCHAR(100) NOT NULL COMMENT '物模型名称',
  `version` VARCHAR(50) NOT NULL DEFAULT '1.0.0' COMMENT '版本号',
  `description` TEXT COMMENT '物模型描述',
  `type` VARCHAR(20) NOT NULL DEFAULT 'custom' COMMENT '类型: custom/template',
  `template_id` BIGINT COMMENT '模板ID',
  `parent_id` BIGINT COMMENT '父版本物模型ID',
  `status` VARCHAR(20) NOT NULL DEFAULT 'draft' COMMENT '状态: draft/published',
  `created_by` BIGINT NOT NULL COMMENT '创建人ID',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  INDEX `idx_type` (`type`),
  INDEX `idx_status` (`status`),
  INDEX `idx_template_id` (`template_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='物模型表';
```

### 2.3 物模型属性表 (thing_model_property)

```sql
CREATE TABLE `thing_model_property` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',

  `enum_values` JSON COMMENT '枚举值列表',
  `unit` VARCHAR(20) COMMENT '单位',
  `access_mode` VARCHAR(20) NOT NULL DEFAULT 'readOnly' COMMENT '读写类型: readOnly/writeOnly/readWrite',
  `default_value` TEXT COMMENT '默认值',
  `description` VARCHAR(500) COMMENT '描述',
  `sort_order` INT DEFAULT 0 COMMENT '排序',
```

## 3. 索引优化建议

| 表名 | 索引 | 说明 |

| device | idx_product_online | (product_id, online_status) |
| device_property | idx_device_identifier_time | (device_id, identifier, timestamp) |

| ota_task_device | idx_task_status | (task_id, status) |

## 4. 分表策略建议

### 4.1 设备属性表分表
- 按device_id哈希分表
- 建议分表数量: 16或32

### 4.2 设备日志表分表
- 按月份分表
- 建议保留6个月数据

### 4.3 设备事件表分表
- 按device_id哈希分表
- 建议分表数量: 16或32
